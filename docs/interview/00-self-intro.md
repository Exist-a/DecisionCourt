> 面试开场白。每次微调，但骨架不变。

## 我的模板（90 秒版）

> "我做 DecisionCourt，**一个 AI 庭审模拟系统**。
>
> **业务侧**：用户提交事实 + 双方立场，系统派出 5 个 AI Agent（控方、辩方、法官、调查员、书记员）模拟完整庭审，最后给出判决。
>
> **技术侧**：这是一个 **AI Agent 编排系统**。我做了 4 件事：
>
> 1. **A2A 消息总线** —— Agent 间不走 prompt 拼接，走显式消息总线，3 种可见性（public / private / team），所有消息落库 → 庭审可回放
> 2. **v0.6 贝叶斯信念引擎** —— AI 法官的"相信度"用贝叶斯 log-odds 数学建模，不是 0-100 主观分，有 belief_diffs 审计 trail
> 3. **v0.7 Gateway 装饰器链** —— 把 LLM 调用的"压缩 / 预算 / 限流 / 降级 / 审计"做成可插拔的装饰器链
> 4. **v0.8 白盒化** —— 让 AI 系统像传统后端一样可观测：三大支柱（结构化日志 / 业务指标 / 业务事件审计）+ 端到端 trace_id 串联
>
> **整个项目最有成就感的是白盒化** —— 交付当天 demo 跑一次，stdout 日志立刻暴露了一个 P1 bug：`llm_calls` 审计表外键约束失败，导致 v0.5 以来 token 成本完全无法统计。5 行代码修复。
>
> **用 4 个词总结**：业务语义 = A2A 总线 = 信念引擎 = 白盒化。"

## 关键名词速查（给面试官不熟时引用）

| 名词 | 我的 1 句话定义 |
|---|---|
| **A2A** | Agent-to-Agent。Agent 间通信的专用总线，跟"直接调 LLM"区别开。 |
| **ReAct** | Reasoning + Acting。LLM 的"思考 → 行动 → 观察 → 再思考"循环。 |
| **Bayesian log-odds** | 贝叶斯对数几率。把"相信度"从 0-100 改成 log(p/(1-p))，数学上可加可减。 |
| **Decorator Pattern** | 装饰器模式。把"附加能力"叠加到核心功能上，不侵入内层。 |
| **Observability 三大支柱** | Logging（结构化日志）/ Metrics（数值指标）/ Tracing（请求级 span）。 |
| **trace_id** | 一次请求的唯一 ID。从 HTTP 入口透传到下游所有调用，全链路可关联。 |

## 【反思】

**关于自我介绍的 3 个我自己的总结**：

1. **不要按时间顺序讲**（"v0.1 做了 X，v0.2 做了 Y..."）—— 面试官 30 秒就忘了。要**按"为什么"逻辑**：业务侧 → 技术侧 → 4 大模块 → 1 个故事。

2. **30 秒电梯版必须熟**。面试官打断"先讲个大概"时，能用 1 段话覆盖全貌。**永远准备 1 个能塞进 60 秒的开场**。

3. **结尾必须是"故事"**（不是技术总结）。技术总结 = 列表（背不下来）。故事 = 有冲突 / 有解决 / 有数字（记得住）。**白盒化发现 P1 bug 这个故事我每次都讲**——3 分钟，有数字（"4 小时 / 1 次 demo / 0 行 → 6 行"），有反思（"业务跑得欢 ≠ 系统健康"）。
