## 🐛 修复清单（9 个静默错误黑洞）

### Backend（4 处静默错误）

| # | 场景 | 修复前 | 修复后 |
|---|------|--------|--------|
| 1 | **opening 死锁**（ReAct max iter）| 黑屏，用户等死 | Toast + 3 recovery 按钮（重启 / 跳过 / 直接判决）|
| 2 | **状态机拒绝**（错阶段操作）| 裸字符串广播，前端无法分类 | `*StateMachineError` typed error + `ClassifyError` 自动拿 phase 拼中文消息 |
| 3 | **WS_THROTTLED**（操作太快）| 裸字符串 `WS_THROTTLED` | `NewUserFacingError(ClassUserInput, CodeActionThrottled)` |
| 4 | **ACTION_FAILED**（操作失败）| `err.Error()` 直接泄露给前端 | `ClassifyError(err)` 自动分类 + retry button |
| 5 | **Trial rate limit 429** | 仅 4 字段 JSON envelope | 4 字段 + `user_facing_error` envelope |

### Frontend（4 处静默错误 + 2 个未捕获）

| # | 场景 | 修复前 | 修复后 |
|---|------|--------|--------|
| 6 | **fetchJson HTTP 4xx/5xx** | `throw new Error("API error: ...")` | `handleApiError` → toast + 抛 `ApiError` |
| 7 | **Verdict 导出失败** | `alert("导出失败")` | `toastFatal` + 页面级 `setExportError` |
| 8 | **auth.ts ensureAuthToken 失败** | 3 处 `console.error` 后返回空 | `reportAuthError` → toast + 清 token |
| 9 | **client component render throw** | 白屏 + dev console | `app/error.tsx` 友好页 + Toast |
| 10 | **root layout throw** | 整个 app 崩 | `app/global-error.tsx` 极简黑白页 |

## 🔄 完整错误链路

```
[后端] 失败发生
   ↓ ClassifyError(err)
   ↓ BroadcastUserFacingError(sessionUUID, ufe)
   ↓ WebSocket event.type === "error"
[前端] handleWsError(payload, { onRecoveryClick })
   ↓ severityFromUFE(class) → level/durationMs
   ↓ useToastStore.push({...})
   ↓ ToastContainer 自动渲染
[用户] 看到 Toast（右下角 / 顶部 banner）
   ↓ 点击 recovery 按钮
[前端] ws.send({ action: r.action, ... })
   ↓ WebSocket user.action
[后端] ProcessUserAction → ForceSkipOpening / RestartOpening
                                                ↑
                            任何 render throw → app/error.tsx
                            任何 root layout throw → app/global-error.tsx
```

## 🎨 4 class 反馈策略

| Class | 展示 | 自动消失 | 按钮 | 典型 code |
|-------|------|----------|------|-----------|
| `user_input` | 右下角 Toast | 3s | 无 | `WS_THROTTLED` / `ACTION_STATE_REJECTED` |
| `transient` | 右下角 Toast | 5s | "重试" | `ACTION_FAILED` / `RECOVERY_FAILED` |
| `degraded` | 顶部 Banner | 不消失 | 自定义 | `BREAKER_DEGRADED` |
| `fatal` | 右下角 Toast | **不消失** | 强制 recovery | `OPENING_SPEECHES_FAILED` / `BUDGET_EXHAUSTED` / `TRIAL_RATE_LIMITED` |

完整 ErrorCode 清单见 [ADR 0024 §2.1](../adr/0024-silent-error-fix-pr1.md#21-类型)。
