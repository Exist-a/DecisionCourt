## 🐛 修复清单

### P0-2: JWT_SECRET 默认值移除（A02 + A05）

**漏洞**：

```go
// backend/internal/config/config.go (修复前)
viper.SetDefault("JWT_SECRET", "decisioncourt-secret")
```

让 `Load()` 的 `mustEnvs` fail-fast 检查形同虚设——用户即使忘了设 JWT_SECRET，程序也照常启动，所有 JWT 都用公开仓库能读到的密钥签发。

**修复**：

```diff
- viper.SetDefault("JWT_SECRET", "decisioncourt-secret")
+ // 不设默认值,缺失时由 Load() 的 mustEnvs log.Fatalf 阻断启动
```

**部署配置配套**：

```yaml
# docker-compose.yml L117 (v0.8.3 已有)
JWT_SECRET: ${JWT_SECRET:?JWT_SECRET must be set in .env}
```

---

### P0-3: 容器 UID 1001 → 10001（A05）

**漏洞**：4 处配置使用 UID 1001，与某些发行版系统用户冲突，且不符合 K8s 部署惯例。

**修复**：

| 文件 | 修改 |
|---|---|
| `backend/Dockerfile` | `addgroup -g 10001 -S appgroup` + `adduser -u 10001 -S appuser -G appgroup` |
| `frontend/Dockerfile` | `addgroup -g 10001 -S nodejs` + `adduser -u 10001 -S nextjs -G nodejs` |
| `docker-compose.yml` backend | `user: "10001:10001"` |
| `docker-compose.yml` frontend | `user: "10001:10001"` |
| `docker-compose.yml` 顶部注释 | `1001:1001` → `10001:10001` |
