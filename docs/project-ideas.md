## 当前项目：决策庭（DecisionCourt）

### 一句话介绍

**决策庭**是一个以"法庭"为隐喻的多 Agent 决策辅助平台。用户面对复杂决策时，可以发起一场"庭审"，由控方、辩方、调查员、书记员等 AI Agent 围绕候选选项进行结构化辩论；用户作为法官/当事人可以实时提交证据、要求调查员搜索、打断质证；最终系统输出一份可执行的《决策判决书》。

### 解决的问题

- 单一 AI 给出的决策建议片面、不可审计
- 复杂决策涉及多维度拉扯，用户难以看清全貌
- 用户容易陷入 confirmation bias，只看自己想看的信息
- 做完重大决定后无法回顾当时的判断依据

### 核心亮点

1. **多 Agent 法庭式辩论**：控方、辩方、调查员、书记员分工协作，不是单模型问答。
2. **用户实时插证据**：用户可以随时提交新证据或要求调查员搜索，Agent 会据此调整策略。
3. **结构化判决书**：输出包含双方主张、证据链、争议焦点、可执行建议的决策报告。
4. **可视化观点地图**：实时展示 Agent 立场、证据影响、立场变化曲线。
5. **Agent Gateway 扩展架构（非 MVP）**：第二阶段引入模型路由、Prompt 压缩、Token 预算、调用审计，实现可量化的成本优化。

### 目标用户

- 职场人：跳槽 vs 留下、创业 vs 稳定
- 产品经理/创业者：产品路线选择、功能优先级
- 投资者：买入/卖出/持有决策
- 学生/研究者：论文选题、研究方向
- 普通人：买房 vs 租房、出国 vs 留下等重大人生决策

| 后端 | Go 1.22+ + Gin + GORM |
| 数据库 | PostgreSQL 15+ + Redis 7+ |
| LLM | DeepSeek-V3 / DeepSeek-R1（OpenAI 兼容格式）|
| WebSearch | SearXNG Docker（开发）+ Tavily API（部署）|
| 实时通信 | WebSocket（gorilla/websocket）|
| 部署 | Docker Compose |
