# 决策庭 - 质证阶段轮次控制修改计划

## 修改概述

将质证阶段从"连续自动循环"改为"用户点击触发每轮"，给予用户充足时间补充证据和思考。

---

## 问题 1：开庭阶段显示"第 0 轮"

**当前状态**：开庭阶段（opening）在 PhaseGuide 中显示"开庭陈述 · 第 0 轮"

**原因**：`transitionPhase` 在进入 opening 时传入 `round=0`

**修改文件**：`backend/internal/courtroom/service.go`

- `StartTrial` 中调用 `transitionPhase(&session, model.PhaseOpening, 0)` 后，不应在 PhaseGuide 显示轮次
- opening 阶段本质是无轮次概念，应显示"开庭陈述"而非"第 0 轮"

**前端配合**：`PhaseGuide` 组件在 `opening` 阶段不显示轮次后缀

---

## 问题 2：结案陈词阶段仍显示质证最后一轮

**当前状态**：closing 阶段复用 `session.CurrentRound`（即质证最后一轮如 3），导致显示"结案陈词 · 第 3 轮"

**原因**：`finishTrial` 调用 `transitionPhase(&session, model.PhaseClosing, session.CurrentRound)` 传入的是质证轮数

**修改文件**：`backend/internal/courtroom/service.go`

- closing 阶段应有独立的轮次计数（如 1），或前端不显示轮次
- 推荐方案：closing 阶段 round 传 0，前端 closing 阶段不显示轮次

---

## 问题 3：质证阶段每轮中间无停顿

**当前状态**：`runCrossExamRound` 是 `for` 循环，后端连续自动执行完所有质证轮次

**目标**：每轮质证完成后暂停，等待用户确认后再开始下一轮

**核心修改**：在 `cross_exam` 每轮结束后，不是自动继续循环，而是引入一个"等待用户触发"的状态

### 后端修改

**文件**：`backend/internal/courtroom/service.go`

1. `runCrossExamRound` 不再是 `for` 循环，每调用一次只执行**一轮**
2. 轮次结束后，广播 `round.waiting_for_user` 事件，前端收到后显示"继续下一轮"按钮
3. 用户点击按钮 → 前端发送 `continue_cross_exam` action → 后端继续下一轮
4. 新增状态 `PhaseWaitingCrossExam` 或复用 `cross_exam` + 前端按钮控制

**推荐方案**：
- 新增 `phase.waiting` 事件，payload 含 `current_round`
- 前端在收到 `phase.waiting` 后显示"开始第 N+1 轮质证"按钮
- 用户点击按钮调用 `POST /api/v1/courtrooms/:uuid/actions` body `{ "action": "continue_cross_exam" }`
- 后端 `ProcessUserAction` 处理 `continue_cross_exam` action，调用 `runCrossExamRound` 执行下一轮

### 前端修改

**文件**：`frontend/components/courtroom/CourtroomScene.tsx`

1. 监听 `round.waiting_for_user` 事件（新增）
2. 在 PhaseGuide 区域下方显示"继续下一轮"按钮
3. 按钮文案：`开始第 {round + 1} 轮质证`
4. 用户点击后调用 `sendAction({ action: "continue_cross_exam" })`

**文件**：`frontend/components/courtroom/PhaseGuide.tsx`

- 考虑在此显示等待提示，或新建专属等待提示区域

---

## 问题 4：PhaseGuide 提示条联动

**当前状态**：PhaseGuide 显示当前阶段和轮次

**修改**：PhaseGuide 需根据新事件类型更新：
- `round.waiting_for_user` 事件时，显示"等待用户确认进入第 N 轮"
- 质证进行中时，显示"第 N 轮质证进行中"
- 用户提交证据后，可显示"证据已提交，等待继续"

---

## 状态转换图（修改后）

```
opening (round=0, 无轮次显示)
    ↓
cross_exam (round=1)
    ↓ 控辩各发言一轮后
广播 round.waiting_for_user (round=1)
    ↓ 用户点击"继续下一轮"
cross_exam (round=2)
    ↓ 控辩各发言一轮后
广播 round.waiting_for_user (round=2)
    ↓ ...
cross_exam (round=3) 或 提前收敛
    ↓ 用户点击或收敛
closing (round=0, 无轮次显示)
    ↓
deliberation → verdict
```

## 实施步骤

### 后端改动

**1. `service.go` - `runCrossExamRound` 改造为单轮执行**

当前逻辑：`for` 循环连续执行直到达到轮数上限或收敛
目标：每调用只执行一轮，结束后广播等待事件

```go
// 修改前：for session.CurrentPhase == model.PhaseCrossExam { ... }
// 修改后：只执行当前轮次，不循环
```

**2. `service.go` - 新增 `round.waiting_for_user` 事件广播**

一轮结束后：
```go
s.broadcastEvent(session.SessionUUID, Event{
    Type: "round.waiting_for_user",
    Payload: map[string]interface{}{
        "current_round": session.CurrentRound,
        "next_round":    session.CurrentRound + 1,
        "max_rounds":    session.MaxRounds,
        "message":       "等待用户确认开始下一轮",
    },
})
```

**3. `service.go` - `ProcessUserAction` 新增 `continue_cross_exam` action**

```go
case "continue_cross_exam":
    return s.runCrossExamRound(ctx, session)
```

**4. `service.go` - 移除 `SubmitEvidence` 中的 `go func` 自动触发**

当前：`SubmitEvidence` 中有 `go func` 自动触发下一轮
修改：用户提交证据后不自动触发下一轮，由用户点击按钮触发

**5. `service.go` - 修复 opening/closing 轮次显示**

- `StartTrial` 进入 opening 时 round=0，但 opening 本身不应显示轮次
- `finishTrial` 进入 closing 时 round=0

### 前端改动

**1. `CourtroomScene.tsx` - 监听 `round.waiting_for_user` 事件**

```typescript
case "round.waiting_for_user": {
    const p = event.payload as {
        current_round: number;
        next_round: number;
        max_rounds: number;
    };
    // 设置等待状态，显示继续按钮
    setWaitingForNextRound(true);
    setNextRoundInfo(p.next_round);
    break;
}
```

**2. `CourtroomScene.tsx` - 显示"继续下一轮"按钮**

在 PhaseGuide 下方或庭审区域显示按钮：
```tsx
{waitingForNextRound && (
    <Button onClick={() => sendAction({ action: "continue_cross_exam" })}>
        开始第 {nextRound} 轮质证
    </Button>
)}
```

**3. `PhaseGuide.tsx` - 质证等待时不显示轮次**

当 `phase === "cross_exam"` 且处于等待状态时，可显示"等待确认"

## 测试场景

- 标准模式（3轮）：每轮之间有用户确认步骤，提交证据后自动继续下一轮
- 快速模式（2轮）：同上
- 深度模式（5轮）：同上
- 提前收敛：信念收敛后直接进入结案，无需用户确认
- 直接判决：用户点击直接判决按钮，跳过剩余轮次
