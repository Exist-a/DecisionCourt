# 庭审可视化简化计划

## Summary

当前庭审页面同时展示两张图表（观点地图 + 立场变化曲线），存在两个问题：

1. **认知负担高**：观点地图使用 ReactFlow 网络图，节点多、连线密集，普通用户难以快速理解
2. **页面拥挤**：两个图表并排（`grid-cols-2`），占用大量垂直空间

**简化方案**：保留 ReactFlow 观点地图但精简节点，删除立场变化曲线组件，腾出页面空间。

## Assumptions & Decisions

| 决策                  | 理由                                      |
| ------------------- | --------------------------------------- |
| 删除证据节点              | 证据数量可能很多（10+），节点密集会让布局混乱；证据影响通过边的线宽已能体现 |
| 删除立场变化曲线            | 用户明确选择"删除"，降低认知负担                       |
| 保留 ReactFlow 框架     | 比静态柱状图更具交互性（可拖拽、缩放）                     |
| 不删除 StanceChart.tsx | 保留文件以便未来恢复，符合"避免破坏性删除"原则                |
| 移除 Background 网格    | 减少视觉噪音，让节点和边更突出                         |
